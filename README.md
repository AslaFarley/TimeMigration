# 《时间移民》(TimeMigration) — 项目总结介绍

## 一、项目概述

**《时间移民》**是一款以刘慈欣同名小说为灵感来源的**科幻文字策略游戏**。玩家作为移民领袖，携带 12 万冷冻人口，在未来的不同时代之间"跳跃迁移"——每次选择一个睡眠时长，跨越数百年乃至上千年，苏醒后观察新时代的面貌，判断是否值得在此定居。核心是对未来**下注**：用手里的冷冻人口和睡眠时长，赌一个能让人类文明延续下去的好时代。

**技术栈**：`React 18` + `TypeScript` + `Vite 5`，纯静态前端项目，桌面浏览器运行。

**当前版本**：`v0.1.0` MVP，完整可玩的核心循环已实现。

---

## 二、核心玩法循环

```
开局：携带 12 万冷冻人口，抵达第一个时代
  ↓
[决策阶段 · 限时 60 秒]
  阅读当前时代的公开叙述（含不可靠信息） + 历史记录 + 先遣队情报
  做出最多 3 个决策：
    ① 是否全体解冻、定居此时代？ → 是：进入结局结算
    ② 选择本次睡眠时长（决定时代跨度与风险）
    ③ 是否派遣先遣队
  ↓
[沉睡过渡] 氛围文字过渡（可兼作 LLM 加载缓冲）
  ↓
[苏醒结算] 世界按状态机演化，揭晓"叙述 vs 真相"
  ↓
回到决策阶段，直到玩家选择定居或触发失败
```

**玩家每轮仅 3 个输入**：是否定居、睡眠多久、是否派先遣队。已解冻人口（`activePop`）的增减完全由世界演化驱动，玩家不能直接操控。

## 三、核心系统设计

### 3.1 五大世界参数

| 参数 | 含义 | 作用 |
|------|------|------|
| **宜居度** (habitability) | 客观环境好坏（天灾维度） | 影响人口乘子 |
| **接纳度** (acceptance) | 原住文明欢迎度（人祸维度） | 影响人口乘子 & 民心 |
| **科技** (tech) | 时代科技水平 | 影响人口乘子 & 承载力 |
| **承载力** (capacity) | 人口容量上限 | 超载触发渐进冲突 |
| **民心** (trust) | 社会稳定度 | 归零触发失败 |

### 3.2 人口演化公式

每次苏醒结算执行三步：
1. **先遣队归队**：`scoutReturn = scoutSent × f(宜居度, 接纳度, 科技)`，注入 `activePop`
2. **已定居人口演化**：`activePop *= f`
3. **长睡损耗**：睡眠时间越长，人口自然损耗越大（建交后减半）

`f` 是环境综合乘子，范围 0.30~1.40：好时代人口壮大，差时代人口衰减。

### 3.3 先遣队机制（scoutForce）

- 每次派遣固定 1200 人（从冷冻人口池扣减）
- **不选任务**：系统自动判断当前对人口存活最关键的参数（接纳度 vs 宜居度），自动改造
- **三重价值**：
  1. 注入人口（存活者并入 `activePop`）
  2. 改造未来（小幅提升下一时代对应参数）
  3. 带回情报（存活率比公开叙述更可靠）

### 3.4 渐进式失败（非瞬间死亡）

- **民心崩溃**：`trust` 低于 25 先出预警（持续数轮），归零才失败
- **人口超载**：`activePop > capacity` → 先警告 → 持续超载才触发冲突损耗
- 避免传统游戏"一个坏随机直接死"的挫败感

## 四、叙述系统（核心创新：把 LLM 不稳定变为 feature）

### 4.1 受控噪声注入

叙述不是"GM 告诉你真相"，而是由一个**有偏见的叙述者**提供：

| 叙述者风格 | 行为 |
|-----------|------|
| **可靠型** (reliable) | 基本如实 |
| **乐观夸大** (optimistic) | 把差的说成好的 |
| **悲观型** (pessimistic) | 把好的说成差的 |
| **撒谎型** (liar) | 故意与真值矛盾 |

代码层持有"真值"，仅对外展示注入偏差的文本。玩家通过多轮苏醒对比、核对历史记录和先遣队情报，**学会识别叙述者风格**——这成为核心玩法。

### 4.2 双重信息来源

- **公开叙述**：有偏差的时代描述
- **先遣队情报**：派过先遣队的时代提供更准的"内部报告"（真实损耗率）
- **历史面板**：每次苏醒记录「当时叙述 vs 实际真值」，可随时回看

### 4.3 LLM 集成（已实装）

- **多 provider 运行时择优**：内置 3 个 OpenAI 兼容接口（SiliconFlow / AiPing / QnAIGC），自动尝试并缓存第一个成功的
- **种子库兜底**：LLM 全部失败时自动回退手写种子库，游戏不崩溃
- **结局 LLM**：成败由规则判定（确定性），但结局文本由 LLM 生成「移民文明苏醒后 1000 年发展史」的史诗叙事

---

## 五、结局系统（开放式，非确定性评分）

- **不给数字评分**，避免"你得了 85 分"的确定性结局感
- 第一版显示：成功/失败/重生 + 规则化定性描述
- LLM 增强：根据最终世界状态 + 整局历史轨迹 + 玩家所有决策，生成一段 **"千年发展史"**叙事，成败由玩家自行解读
- 从第一版起**完整记录历史轨迹**，确保未来切 LLM 时上下文数据现成

## 六、时代库（32 种时代类型）

远超传统"10 种年代"设计，涵盖：

黄金时代 / 战乱 / 生态崩溃 / 科技奇点 / 伪繁荣 / 寒冬 / 瘟疫 / 复兴 / 极权 / 真空荒芜 / 原始地球 / 沙漠扩张 / 洪水避难 / 游牧复兴 / 实验室飞地 / 数字乌托邦 / 铁腕专制 / 邪教觉醒 / 贸易复兴 / 海上定居 / 地下城市 / 太阳朋克 / 基因分化 / 信息黑洞 / 地球化黎明 / 新封建 / 海洋崩溃 / 遗忘巨碑 / 回声室 / 核秋天 / 冰川行者 / 巨构废墟 / 星辰远航

每种类型有独特的参数倾向（如"伪繁荣" = 叙述乐观但真实信任度/宜居度偏低）。

## 八、技术亮点

1. **受控噪声作为玩法**：叙述不可靠不是 bug，是核心设计
2. **严格分离真值与展示**：代码永远持有真实数值，仅展示层注入偏差
3. **LLM 多 provider 容错**：3 路 LLM + 种子库兜底，游戏永不崩溃
4. **纯同步 reducer + 异步编排**：数值演化保持确定性，LLM 文本异步覆盖
5. **完整历史轨迹**：每轮详细记录，为未来 LLM 上下文做好准备
6. **渐进式失败**：预警→缓冲→失败，非瞬间死亡
